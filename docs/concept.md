# Концепция модульной сортировочной системы

## Назначение системы

Система предназначена для автоматической сортировки товаров в сортировочном центре.

Она должна принимать поток товаров, считывать штрихкоды, получать направление от внешней системы управления складом и доставлять каждый товар в нужный картонный тарный ящик.

## Базовая идея

Предлагается использовать модульную двухступенчатую сортировочную систему.

Общий поток товаров сначала распределяется между несколькими параллельными линиями. Затем товар проходит сканирование и первичную сортировку по группе направлений. После этого он поступает во вторую ступень сортировки, где направляется к конкретному выходному контейнеру.

Такая схема позволяет:

* распределить нагрузку между несколькими линиями;
* уменьшить длину одной сортировочной линии;
* обслуживать большое количество направлений;
* отключать отдельные модули без полной остановки системы;
* масштабировать систему добавлением новых линий и модулей.

## Основные элементы системы

В состав системы входят:

* зона подачи товаров;
* параллельные входные линии;
* датчики наличия товара;
* сканеры штрихкодов;
* контроллер управления;
* имитация внешней WMS;
* модули первичной сортировки;
* промежуточные буферы;
* модули вторичной сортировки;
* выходные картонные тарные ящики;
* система сбора статистики.

## Путь товара

1. Товар поступает в зону подачи.
2. Система направляет его на одну из доступных входных линий.
3. Датчик фиксирует появление товара.
4. Сканер считывает штрихкод.
5. Контроллер отправляет запрос в имитацию WMS.
6. WMS возвращает целевое направление.
7. Алгоритм выбирает доступный маршрут.
8. Товар проходит первичную сортировку по группе направлений.
9. При необходимости товар временно помещается в буфер.
10. На второй ступени товар направляется к конкретному выходу.
11. Датчик подтверждает успешный сброс.
12. Система обновляет заполненность выходного контейнера.
13. При заполнении контейнера направление временно блокируется до замены тары.

## Обработка нештатных ситуаций

Если штрихкод не считан, товар направляется на повторное сканирование или на отдельный участок ручной обработки.

Если один из модулей перегружен, товар временно задерживается в буфере или перенаправляется по резервному маршруту.

Если отдельная линия выходит из строя, поток перераспределяется между оставшимися линиями.

Если выходной контейнер заполнен, новые товары временно не направляются в этот выход до завершения замены тары.

## Масштабирование

Система должна быть модульной.

Производительность можно увеличивать следующими способами:

* добавлением новых входных линий;
* добавлением сортировочных модулей;
* увеличением количества буферов;
* разделением направлений между несколькими зонами;
* запуском нескольких одинаковых сортировочных контуров.

## Цель симуляции

Python-симуляция должна проверить:

* достижимую производительность;
* среднее время прохождения товара;
* длину очередей;
* загрузку линий;
* загрузку буферов;
* количество заторов;
* количество ошибок сканирования;
* поведение системы при отказах;
* влияние количества линий на общую производительность.
